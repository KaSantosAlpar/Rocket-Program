MoodTrack
Senhores e senhoritas do Rocket Sul, bom dia!!
 
Bem vindos ao último projeto do Módulo 1! E nós vamos unir o útil ao agradável! 
 
A Thais ensinou até o momento soft skills muito importantes pra vocês, então para ajudar a praticar essas skills, vocês vão criar um monitor de hábitos e diário de emoções!
 
O objetivo deste projeto é desenvolver uma aplicação web interativa que permita ao usuário registrar seus hábitos diários e também acompanhar seu estado emocional ao longo do tempo.
A aplicação deverá permitir que o usuário registre informações diárias, visualize seus registros e acompanhe pequenas análises sobre seu comportamento e humor.
Este projeto foi pensado para aplicar na prática conceitos de JavaScript avançado e arquitetura de aplicações web, incluindo orientação a objetos, organização modular do código e uso de AngularJS. (Basicamente, usem tudo o que vocês aprenderam até agora)
 
A aplicação deverá permitir que vocês: 
Registrem os hábitos realizados no dia
Registrem seu estado emocional
Visualizem registros anteriores
Filtrem registros
Visualizem pequenos indicadores de progresso
 
Funcionalidades obrigatórias:
 
Registro de Humor
O sistema deve permitir que o usuário registre seu humor diário.
Campos obrigatórios:
Data
Nível de humor (escala de 1 a 4)
Observação opcional

Exemplo de níveis:
1 – Muito triste

2 – Neutro

3 – Feliz

4 – Muito feliz
(os níveis deverão ser convertidos visualmente em emojis)
--------------------------------------------------------------------------
 
Registro de Hábitos
O usuário deve poder registrar hábitos realizados no dia.
Cada hábito deve possuir:
Nome do hábito
Status (concluído ou não concluído)

Exemplos de hábitos:
Beber água
Exercício
Estudo
Leitura
Dormir cedo
----------------------------------------------------------------------------------------------
 
Lista de Registros
A aplicação deve exibir uma lista com os registros realizados.
Cada registro deve mostrar:
data
humor
hábitos registrados
observação
---------------------------------------------------------------------------------------------
 
Filtros
A aplicação deve permitir filtrar registros por:
data
nível de humor
